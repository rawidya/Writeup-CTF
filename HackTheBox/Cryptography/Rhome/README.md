# HTB Write-up: Rhome

## 📝 Challenge Summary
**Rhome** is a cryptography challenge involving a custom Diffie-Hellman (DH) key exchange. While the modulus is large, the generator is maliciously chosen to confine the protocol to a dangerously small subgroup, making the Discrete Logarithm Problem (DLP) trivial to solve.

---

## 🔎 Recon (The Subgroup Vulnerability)
The server generates prime parameters using the formula `p = 2 * q * r + 1`, where `q` is a small 42-bit prime and `r` is a large 512-bit prime.
The generator `g` is calculated as:
$g = h^{2r} \pmod{p}$

### Mathematical Proof
According to Fermat's Little Theorem, $h^{p-1} \equiv 1 \pmod{p}$.
Since $p-1 = 2qr$, then $h^{2qr} = (h^{2r})^q = g^q \equiv 1 \pmod{p}$.
This proves that the order of the subgroup generated by $g$ is exactly **$q$**.

In a standard DH exchange, the order should be a large prime to prevent brute-force attacks. Here, because $q$ is only 42 bits, we can solve the secret key $a \pmod{q}$ in minutes.

## ⚙️ Strategy: Solving the Discrete Log
Since the public keys $A$ and $B$ are restricted to this small subgroup, we can find the private exponent $a'$ such that $g^{a'} \equiv A \pmod{p}$ using the **Baby-step Giant-step (BSGS)** algorithm.

### Attack Steps
1.  **Connect**: Retrieve public parameters $p, g, A, B$.
2.  **Factor**: Find $q$ by factoring $(p-1)/2$.
3.  **Solve**: Run BSGS in the subgroup of order $q$ to find the private key $a'$.
4.  **Shared Secret**: Calculate $SS = B^{a'} \pmod{p}$.
5.  **Decrypt**: Derive the AES key from the shared secret and decrypt the flag.
